Formularios HTML y validación: recopila datos de forma accesible

Publicado por ByteNova · 22 de julio de 2026

En este artículo aprenderás:

Aprende a crear formularios accesibles con etiquetas, tipos de campo y errores claros.

Los formularios permiten contactar, registrarse, buscar o reservar. Deben ser comprensibles, usables con teclado y resistentes a datos inesperados.

¿Qué es?

Un formulario HTML agrupa controles para recopilar información. Cada control necesita nombre, etiqueta asociada y validación adecuada.

Idea clave

Aprende a crear formularios accesibles con etiquetas, tipos de campo y errores claros.

¿Por qué es importante?

Una estructura correcta reduce errores, mejora la experiencia móvil y permite que tecnologías de asistencia anuncien cada campo.

Comprender este fundamento permite tomar mejores decisiones, reconocer límites y relacionar la teoría con situaciones reales. La práctica debe centrarse en problemas pequeños antes de aumentar la complejidad.

Conceptos principales

label

Describe el propósito y se asocia con for e id.

name

Clave con la que se envía el dato.

type

Adapta el control al dato.

required

Indica que debe completarse.

fieldset

Agrupa opciones relacionadas.

Error

Explica qué corregir.

Solicita solo datos necesarios.

4

Campo de correo




Usaremos tu correo para responder.

La etiqueta permanece visible y el tipo ayuda en móvil.

¿Dónde se utiliza?

  • Contacto
  • Registro
  • Reservas
  • Buscadores
  • Encuestas

En proyectos reales suele combinarse con otros conceptos. Conviene observar qué problema resuelve, qué datos necesita y cómo comprobar que el resultado es correcto.

Buenas prácticas

  • Pide pocos datos.
  • Mantén etiquetas visibles.
  • Usa autocomplete.
  • Explica errores cerca del campo.
  • Conserva foco visible.
Practica con criterio

No memorices reglas aisladas. Comprende el propósito, prueba casos distintos y revisa el resultado antes de usarlo con información importante.

Errores frecuentes

  • Usar placeholder como etiqueta.
  • Desactivar pegar.
  • Mostrar errores vagos.
  • Validar solo con JavaScript.
  • Borrar datos válidos.

Etiquetas, tipos de campo y mensajes útiles

Un formulario accesible relaciona cada control con una etiqueta visible mediante for e id. El atributo name identifica el dato enviado, mientras que el tipo de campo ayuda al navegador a mostrar el teclado correcto y aplicar validaciones básicas. El texto de ejemplo dentro de placeholder no sustituye una etiqueta porque desaparece al escribir.

La validación del navegador mejora la experiencia, pero no reemplaza la validación en el servidor. Cualquier dato recibido debe considerarse no confiable. Los mensajes deben explicar qué ocurrió y cómo corregirlo; “dato inválido” es menos útil que “escribe un correo con formato nombre@dominio.com”.

<label for="correo">Correo electrónico</label>
<input id="correo" name="correo" type="email"
       autocomplete="email" required
       aria-describedby="ayuda-correo">
<small id="ayuda-correo">Usaremos este correo para responder.</small>

Práctica guiada: formulario de contacto

Construye campos para nombre, correo, asunto y mensaje. Añade límites razonables y un aviso de estado después del envío. Prueba el formulario sin ratón, con campos vacíos y con formatos incorrectos.

  • No solicites información que no necesitas.
  • Agrupa opciones relacionadas con fieldset y legend.
  • No dependas únicamente del color para señalar errores.
  • Conserva lo escrito cuando una validación falle.

Cómo empezar a practicar

  1. Crea nombre, correo y mensaje.
  2. Asocia labels.
  3. Agrega tipos.
  4. Prueba campos vacíos.
  5. Navega con teclado.

Conserva cada ejercicio y escribe una breve nota sobre la decisión que tomaste. Después crea una segunda versión más clara o más resistente a errores.

Preguntas frecuentes

¿required reemplaza al servidor?

No. El servidor debe validar.

¿Por qué no basta placeholder?

Desaparece y puede tener poco contraste.

¿Qué hace autocomplete?

Ayuda a completar datos conocidos.

¿Cómo mostrar errores?

Con texto específico y asociado.

¿Qué botón envía?

Un button type="submit".
