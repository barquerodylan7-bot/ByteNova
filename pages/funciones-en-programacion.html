Funciones en programación: cómo organizar y reutilizar código

Publicado por ByteNova · 22 de julio de 2026

En este artículo aprenderás:

Descubre cómo una función recibe parámetros, devuelve resultados y organiza el código.

Cuando un programa crece, mezclar todas las instrucciones dificulta entenderlo. Las funciones agrupan una responsabilidad bajo un nombre.

¿Qué es?

Una función es un bloque reutilizable que realiza una tarea concreta, puede recibir parámetros y devolver un resultado.

Idea clave

Descubre cómo una función recibe parámetros, devuelve resultados y organiza el código.

¿Por qué es importante?

Las funciones reducen duplicación, facilitan las pruebas y permiten modificar una regla en un solo lugar.

Comprender este fundamento permite tomar mejores decisiones, reconocer límites y relacionar la teoría con situaciones reales. La práctica debe centrarse en problemas pequeños antes de aumentar la complejidad.

Conceptos principales

Definición

Describe nombre, parámetros e instrucciones.

Llamada

Ejecuta la función.

Parámetro

Nombre de un dato recibido.

Argumento

Valor enviado al llamar.

Retorno

Resultado que entrega.

Calcular un precio

def calcular_total(precio, impuesto):
    return precio + precio * impuesto

total = calcular_total(100, 0.13)
print(total)

La fórmula puede reutilizarse con valores distintos.

¿Dónde se utiliza?

  • Validaciones
  • Cálculos
  • Transformación de datos
  • Eventos de interfaz
  • Acceso a servicios

En proyectos reales suele combinarse con otros conceptos. Conviene observar qué problema resuelve, qué datos necesita y cómo comprobar que el resultado es correcto.

Buenas prácticas

  • Crea funciones pequeñas.
  • Usa nombres descriptivos.
  • Evita variables globales.
  • Devuelve valores.
  • Prueba casos distintos.
Practica con criterio

No memorices reglas aisladas. Comprende el propósito, prueba casos distintos y revisa el resultado antes de usarlo con información importante.

Errores frecuentes

  • Hacer demasiadas tareas.
  • Usar parámetros innecesarios.
  • Modificar estado externo sin avisar.
  • Repetir funciones parecidas.
  • Ignorar resultados vacíos.

Parámetros, retorno y responsabilidad única

Una función recibe información mediante parámetros y puede devolver un resultado. La diferencia entre imprimir y retornar es importante: imprimir solo muestra algo en pantalla, mientras que retornar permite guardar el valor, combinarlo con otros cálculos o probarlo de forma automática. Una función útil debe tener un nombre que describa la acción y una responsabilidad concreta.

Cuando una función necesita demasiados parámetros o realiza validación, cálculo, almacenamiento y presentación al mismo tiempo, probablemente está haciendo demasiado. Dividirla en funciones pequeñas facilita localizar errores y reutilizar partes sin copiar código. También permite escribir pruebas para cada comportamiento.

def calcular_descuento(precio, porcentaje):
    if precio < 0 or porcentaje < 0:
        raise ValueError("Los valores no pueden ser negativos")
    return precio * porcentaje / 100

ahorro = calcular_descuento(25000, 10)

La función calcula y retorna; otra parte del programa decide cómo mostrar o guardar el resultado.

Práctica guiada: calculadora modular

Crea funciones independientes para sumar, restar, multiplicar y dividir. La división debe manejar el caso de divisor igual a cero. Después construye una función que reciba el nombre de la operación y llame a la función adecuada.

  • Prueba cada función con números positivos, negativos y cero.
  • Evita variables globales; pasa los datos como parámetros.
  • Escribe una frase corta que documente qué recibe y qué devuelve.
  • Compara la solución con una versión que tenga todo el código mezclado.

Cómo empezar a practicar

  1. Convierte un cálculo repetido.
  2. Crea dos parámetros.
  3. Devuelve un resultado.
  4. Prueba tres casos.
  5. Divide un programa largo.

Conserva cada ejercicio y escribe una breve nota sobre la decisión que tomaste. Después crea una segunda versión más clara o más resistente a errores.

Preguntas frecuentes

¿Siempre devuelve algo?

No, aunque devolver valores facilita reutilizar y probar.

¿Parámetro y argumento son iguales?

El parámetro se define; el argumento se envía.

¿Cuántos parámetros?

Solo los necesarios.

¿Qué es una función pura?

Depende de entradas y no modifica estado externo.

¿Por qué facilita pruebas?

Cada responsabilidad puede comprobarse por separado.

Conclusión

Una función es un bloque reutilizable que realiza una tarea concreta, puede recibir parámetros y devolver un resultado.

Las funciones reducen duplicación, facilitan las pruebas y permiten modificar una regla en un solo lugar.
