Cuando aparecen usuarios, pedidos y productos, una aplicación necesita una estructura confiable y consistente.

¿Qué es?

Una base relacional almacena información en tablas. Las claves identifican filas y conectan entidades sin repetir todos sus datos.

Practica con criterio

No memorices reglas aisladas. Comprende el propósito, prueba casos distintos y revisa el resultado antes de usarlo con información importante.

Errores frecuentes

  • Varios valores en una columna.
  • Nombres ambiguos.
  • Eliminar sin estrategia.
  • Indexar todo.
  • Ignorar transacciones.

Claves, relaciones y normalización básica

Una clave primaria identifica cada fila de forma única. Una clave foránea guarda la referencia a una fila de otra tabla y permite expresar relaciones. Por ejemplo, cada artículo puede pertenecer a una categoría mediante categoria_id, sin repetir el nombre completo de la categoría en todos los registros.

La normalización busca reducir duplicación y anomalías. Si el correo de un autor aparece en cien artículos, corregirlo en cien lugares es propenso a errores. Separar autores y artículos permite actualizarlo una vez. Sin embargo, el diseño debe responder a necesidades reales: crear demasiadas tablas sin propósito también vuelve difícil el sistema.

categorias(id, nombre)
articulos(id, titulo, categoria_id)

articulos.categoria_id → categorias.id

Práctica guiada: modelo para ByteNova

Dibuja tablas para artículos, categorías, etiquetas y autores. Un artículo pertenece a una categoría, pero puede tener varias etiquetas. Para esa relación de muchos a muchos, crea una tabla intermedia con los identificadores del artículo y la etiqueta.

  • Define una clave primaria para cada tabla.
  • Marca qué campos no pueden quedar vacíos.
  • Decide qué debe ocurrir si se elimina una categoría.
  • Agrega datos de ejemplo y comprueba que no necesites repetir información.

Cómo empezar a practicar

  1. Dibuja entidades.
  2. Define claves.
  3. Crea una relación.
  4. Escribe filas.
  5. Busca duplicaciones.

Conserva cada ejercicio y escribe una breve nota sobre la decisión que tomaste. Después crea una segunda versión más clara o más resistente a errores.

Preguntas frecuentes

¿Qué significa relacional?

Tablas conectadas mediante claves.

¿Qué es normalización?

Reducir duplicación y dependencias.

¿Qué es una clave foránea?

Una referencia a otra tabla.

¿Qué es una transacción?

Operaciones que se confirman o deshacen juntas.

¿Todas las bases son relacionales?

No, existen otros modelos.
