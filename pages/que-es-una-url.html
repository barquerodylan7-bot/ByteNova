¿Qué es una URL y cuáles son sus partes? 🔗

Publicado por ByteNova · 17 de julio de 2026 · Actualizado el 21 de julio de 2026

En este artículo aprenderás:

Qué es una URL, cómo permite localizar recursos en Internet, cuáles son sus partes y qué diferencias existen entre rutas absolutas, relativas, parámetros y fragmentos.

Una URL es la dirección utilizada para localizar una página, imagen, archivo, video u otro recurso disponible dentro de Internet.

Cuando escribes una dirección en el navegador o presionas un enlace, estás utilizando una URL para indicar qué recurso deseas abrir.

¿Qué es una URL?

URL son las siglas de Uniform Resource Locator , que en español puede traducirse como localizador uniforme de recursos.

Su función consiste en indicar dónde se encuentra un recurso y qué método debe utilizarse para acceder a él.

Ejemplo de una URL 
https://bytenova.app/pages/que-es-una-url.html

En este ejemplo, la dirección indica que el navegador debe utilizar HTTPS, conectarse con el dominio de ByteNova y solicitar un archivo determinado.

¿Para qué sirve una URL?

  • Abrir páginas web.
  • Localizar imágenes y videos.
  • Descargar documentos.
  • Acceder a archivos CSS y JavaScript.
  • Enviar información mediante parámetros.
  • Dirigir al usuario hacia una sección específica.
  • Compartir contenido mediante enlaces.
  • Conectar páginas dentro de un sitio.
  • Identificar recursos utilizados por una aplicación.
Ejemplo sencillo

Una página puede tener una URL para su portada, otra para cada artículo y otras para sus imágenes, documentos o archivos de estilos.

Partes principales de una URL

Una URL puede contener diferentes elementos. No todas las direcciones necesitan utilizar cada una de estas partes.

URL de ejemplo 
https://www.ejemplo.com:443/articulos/html?categoria=web#introduccion
Protocolo

https://

Subdominio

www

Dominio

ejemplo.com

Puerto

:443

Ruta

/articulos/html

Parámetro

?categoria=web

Fragmento

#introduccion

¿Qué es el protocolo de una URL?

El protocolo aparece al inicio e indica cómo debe accederse al recurso.

HTTP

Se utiliza para transferir información web sin la protección moderna proporcionada por HTTPS.

HTTPS

Utiliza cifrado para proteger la comunicación entre el navegador y el servidor.

FTP

Puede utilizarse para transferir archivos entre sistemas.

mailto

Permite crear enlaces que intentan abrir una aplicación de correo.

Ejemplo de enlace de correo 
<a href="mailto:contacto@ejemplo.com">
    Enviar correo
</a>

Puedes ampliar este tema en el artículo sobre HTTP y HTTPS .

Dominio y subdominio

El dominio identifica el sitio o servicio al que deseas acceder.

Dominio de ByteNova 
bytenova.app

El sistema DNS permite relacionar ese nombre con la dirección del servidor correspondiente.

¿Qué es un subdominio?

Un subdominio es una sección que aparece antes del dominio principal. 

blog.ejemplo.com
tienda.ejemplo.com
soporte.ejemplo.com

Puede utilizarse para separar partes de un proyecto, aplicaciones o servicios diferentes.

Revisa el dominio completo

En una dirección como bytenova.app.sitio-falso.com , el dominio principal sería sitio-falso.com , no ByteNova.

Aprende más en el artículo sobre dominio y hosting .

¿Qué es la ruta de una URL?

La ruta aparece después del dominio e indica el recurso o sección que debe solicitarse.

Ejemplo 
https://bytenova.app/pages/articulos.html

En esta URL, la ruta es: 

/pages/articulos.html

En un sitio estático, esta ruta puede coincidir con la ubicación real del archivo dentro de las carpetas del proyecto.

Las mayúsculas pueden importar

En muchos servidores, Articulo.html y articulo.html pueden considerarse archivos diferentes.

¿La URL debe mostrar una extensión?

No necesariamente. Una dirección puede terminar en .html o utilizar una estructura sin mostrar la extensión del archivo. 

https://ejemplo.com/articulos/que-es-html

El resultado depende de la configuración utilizada por el servidor o la plataforma de publicación.

¿Qué son los parámetros de consulta?

Los parámetros permiten enviar información adicional mediante una URL.

Comienzan después del símbolo de interrogación: 

https://ejemplo.com/buscar?tema=html

En este ejemplo:

  • tema es el nombre del parámetro.
  • html es su valor.

Varios parámetros

Los parámetros adicionales se separan mediante el símbolo & . 

https://ejemplo.com/buscar?tema=html&nivel=principiante
No coloques información sensible en la URL

Las direcciones pueden quedar guardadas en el historial, registros del servidor, herramientas de análisis y enlaces compartidos.

¿Qué es un fragmento?

Un fragmento permite dirigir al usuario hacia una parte específica de una página.

Se identifica mediante el símbolo # .

Ejemplo 
https://ejemplo.com/articulo.html#conclusion

Para que funcione, debe existir un elemento con el identificador correspondiente: 

<h2 id="conclusion">
    Conclusión
</h2>

Los índices de los artículos de ByteNova utilizan este sistema para llevar al lector hacia cada sección.

URL absolutas y relativas

URL absoluta

Incluye la dirección completa, comenzando por el protocolo y el dominio.

URL relativa

Indica una ubicación tomando como referencia la página actual.

Ejemplo de URL absoluta 

https://bytenova.app/pages/articulos.html

Ejemplo de URL relativa 

articulos.html

Cuando ambos archivos están dentro de la misma carpeta, puede utilizarse solamente el nombre del archivo.

Subir una carpeta

Los dos puntos seguidos de una barra permiten retroceder un nivel dentro de la estructura: 

../index.html

En los artículos de ByteNova, esta ruta permite volver desde la carpeta pages hacia el archivo principal del proyecto.

Ejemplos de rutas relativas 
articulos.html
../index.html
../css/style.css
../images/favicon.svg

¿Cuándo utilizar cada tipo?

  • Utiliza rutas relativas para archivos internos del proyecto.
  • Utiliza URL absolutas para enlaces externos.
  • Utiliza direcciones absolutas en enlaces canónicos.
  • Revisa las rutas cuando muevas archivos de carpeta.
  • Evita mezclar dominios antiguos y nuevos.

URL y posicionamiento SEO

Una URL clara puede ayudar a las personas y a los buscadores a comprender el contenido de una página.

URL clara

/pages/que-es-una-url.html

URL poco descriptiva

/pages/documento29x7.html

Buenas prácticas para crear URL

  • Utilizar palabras relacionadas con el contenido.
  • Mantenerlas fáciles de leer.
  • Separar palabras mediante guiones.
  • Evitar caracteres innecesarios.
  • Evitar cambiar direcciones sin necesidad.
  • Utilizar una sola versión oficial de la página.
  • Mantener una estructura consistente.
  • Evitar parámetros innecesarios.
Utiliza guiones entre palabras 
que-es-una-url.html

Es preferible a estructuras como: 

que_es_una_url.html
queesunaurl.html

Puedes ampliar estas recomendaciones en el artículo sobre qué es el SEO .

¿Qué es una URL canónica?

Una URL canónica indica cuál es la dirección principal que debe considerarse como versión oficial de una página. 

<link
    rel="canonical"
    href="https://bytenova.app/pages/que-es-una-url.html"
>

Esto ayuda cuando un mismo contenido puede encontrarse mediante más de una dirección.

Cómo identificar una URL sospechosa

Los atacantes pueden crear direcciones parecidas a las de servicios legítimos para intentar robar información.

  • Dominios con letras cambiadas.
  • Subdominios que intentan imitar una empresa.
  • Direcciones excesivamente largas.
  • Errores ortográficos en el dominio.
  • Enlaces acortados cuyo destino no es visible.
  • Parámetros extraños o innecesarios.
  • Uso de direcciones IP en lugar del dominio esperado.
  • Mensajes que generan urgencia o miedo.
Ejemplo educativo 
https://cuenta-banco.ejemplo-falso.com

Aunque aparezca la palabra “banco”, el dominio principal es ejemplo-falso.com .

HTTPS no garantiza que una página sea legítima

Una página fraudulenta también puede utilizar HTTPS. Debes revisar cuidadosamente el dominio completo.

Aprende a reconocer este tipo de engaño en el artículo sobre qué es el phishing .

Recomendaciones antes de abrir un enlace

  • Revisa el dominio completo.
  • Busca letras cambiadas o caracteres extraños.
  • No confíes únicamente en el texto visible del enlace.
  • Evita ingresar datos desde mensajes inesperados.
  • Abre directamente la página oficial cuando tengas dudas.
  • No ignores las advertencias del navegador.

Errores frecuentes con las URL

  • Escribir incorrectamente el nombre del archivo.
  • Utilizar una ruta correspondiente a otra carpeta.
  • Confundir mayúsculas y minúsculas.
  • Olvidar la extensión del archivo.
  • Dejar espacios dentro de la dirección.
  • Enlazar hacia un dominio antiguo.
  • Eliminar una página sin actualizar sus enlaces.
  • Utilizar una URL canónica incorrecta.
  • Copiar parámetros innecesarios.
  • Cambiar nombres de archivos sin actualizar el sitemap.
Revisa siempre desde la página publicada

Una ruta puede funcionar dentro de una carpeta local y fallar después de publicar el sitio si utiliza nombres o ubicaciones incorrectas.

Preguntas frecuentes

¿URL y dominio son lo mismo?

No. El dominio identifica el sitio. La URL puede incluir además protocolo, ruta, parámetros y fragmentos.

¿Una URL siempre comienza con HTTPS?

No. Puede utilizar diferentes protocolos, aunque HTTPS es el estándar recomendado para páginas web.

¿Qué diferencia existe entre URL y enlace?

La URL es la dirección del recurso. Un enlace es un elemento que permite abrir esa dirección.

¿Puedo cambiar la URL de una página?

Sí, pero los enlaces anteriores dejarán de funcionar si no se configura una redirección y no se actualizan las referencias internas.

¿Los parámetros forman parte de la URL?

Sí. Permiten enviar información adicional después del símbolo de interrogación.

Conclusión

Una URL es la dirección que permite localizar una página, archivo u otro recurso dentro de Internet.

Puede incluir protocolo, dominio, puerto, ruta, parámetros de consulta y fragmentos.

Comprender su estructura ayuda a crear enlaces correctos, mejorar el SEO, organizar un sitio y reconocer direcciones potencialmente fraudulentas.
