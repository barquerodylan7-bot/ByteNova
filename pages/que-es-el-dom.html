¿Qué es el DOM y cómo JavaScript modifica una página?

Publicado por ByteNova · 22 de julio de 2026

En este artículo aprenderás:

Comprende cómo el DOM representa el HTML y permite que JavaScript modifique una página.

📑 Contenido del artículo

El navegador convierte el HTML en una representación que JavaScript puede consultar y cambiar para crear menús, filtros y validaciones.

¿Qué es?

El DOM es un árbol de nodos que representa elementos, atributos y textos como objetos accesibles desde JavaScript.

Idea clave

Comprende cómo el DOM representa el HTML y permite que JavaScript modifique una página.

¿Por qué es importante?

Conecta la estructura HTML con el comportamiento de la interfaz y ayuda a crear interacción sin recargar toda la página.

Comprender este fundamento permite tomar mejores decisiones, reconocer límites y relacionar la teoría con situaciones reales. La práctica debe centrarse en problemas pequeños antes de aumentar la complejidad.

Conceptos principales

Documento

Punto de entrada principal.

Nodo

Elemento, texto o comentario.

Selector

Localiza elementos.

Atributo

Información declarada en HTML.

Evento

Señal como click o submit.

Cómo aplicarlo paso a paso

1

Selecciona

Usa querySelector.

2

Lee

Consulta texto, clases o valores.

3

Escucha

Registra un evento.

4

Actualiza

Cambia solo lo necesario.

5

Mantén accesibilidad

Sincroniza foco y ARIA.

Cambiar un mensaje

const boton = document.querySelector("#saludar");
const mensaje = document.querySelector("#mensaje");
boton.addEventListener("click", () => {
  mensaje.textContent = "¡Hola!";
});

El evento actualiza únicamente el texto.

¿Dónde se utiliza?

  • Menús
  • Formularios
  • Filtros
  • Acordeones
  • Contadores

En proyectos reales suele combinarse con otros conceptos. Conviene observar qué problema resuelve, qué datos necesita y cómo comprobar que el resultado es correcto.

Buenas prácticas

  • Usa selectores estables.
  • Evita eventos inline.
  • Actualiza poco DOM.
  • Sincroniza ARIA.
  • Conserva foco visible.
Practica con criterio

No memorices reglas aisladas. Comprende el propósito, prueba casos distintos y revisa el resultado antes de usarlo con información importante.

Errores frecuentes

  • Ejecutar antes de cargar.
  • Usar innerHTML con datos no confiables.
  • Duplicar escuchadores.
  • Modificar estilos uno a uno.
  • Olvidar teclado.

Nodos, selección y eventos

Cuando el navegador interpreta HTML crea una estructura de nodos. JavaScript puede buscar nodos con métodos como querySelector, leer o cambiar propiedades y responder a eventos. El DOM no es el archivo HTML original: es la representación activa que existe en memoria y puede cambiar mientras la página está abierta.

Modificar el DOM debe hacerse con cuidado. Repetir búsquedas dentro de un bucle, reconstruir grandes secciones sin necesidad o insertar texto de origen desconocido como HTML puede afectar rendimiento o seguridad. Para texto proporcionado por una persona, textContent suele ser más seguro que innerHTML.

const boton = document.querySelector("#mostrar");
const mensaje = document.querySelector("#mensaje");

boton.addEventListener("click", () => {
  mensaje.textContent = "Contenido actualizado";
  mensaje.hidden = false;
});

Práctica guiada: lista interactiva

Crea un formulario con un campo y un botón. Al enviarlo, agrega un elemento a una lista sin recargar la página. Impide valores vacíos, limpia el campo y permite eliminar cada elemento. Usa un único escuchador en la lista para practicar delegación de eventos.

  • Selecciona los elementos una vez y guarda las referencias.
  • Usa createElement y textContent.
  • Prueba el formulario con teclado, incluida la tecla Enter.
  • Muestra un mensaje cuando la lista esté vacía.

Cómo empezar a practicar

  1. Selecciona un título.
  2. Cambia una clase.
  3. Escucha un clic.
  4. Crea una lista.
  5. Valida un formulario.

Conserva cada ejercicio y escribe una breve nota sobre la decisión que tomaste. Después crea una segunda versión más clara o más resistente a errores.

Preguntas frecuentes

¿Es lo mismo que HTML?

No. HTML es el texto; DOM es la representación.

¿Qué hace querySelector?

Devuelve el primer elemento coincidente.

¿textContent o innerHTML?

textContent es más seguro para texto.

¿Qué es un evento?

Una acción detectada por el navegador.

¿Afecta rendimiento?

Muchos cambios grandes pueden hacerlo.

Conclusión

El DOM es un árbol de nodos que representa elementos, atributos y textos como objetos accesibles desde JavaScript.
