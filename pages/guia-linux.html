Linux para principiantes: guía desde cero 🐧

Qué es Linux, cómo se organiza su sistema de archivos, cuáles son sus distribuciones y cómo utilizar comandos básicos para administrar archivos, programas, permisos, procesos y conexiones.

Linux es una familia de sistemas operativos utilizada en computadoras personales, servidores, dispositivos, sistemas de desarrollo y servicios de Internet.

Puede utilizarse mediante una interfaz gráfica similar a otros sistemas, pero también ofrece una terminal muy potente para ejecutar tareas mediante comandos.

¿Qué es Linux?

Linux es el núcleo utilizado por diferentes sistemas operativos conocidos como distribuciones.

Una distribución combina el núcleo Linux con programas, herramientas, una interfaz gráfica y un sistema para instalar aplicaciones.

Explicación sencilla

Linux administra el funcionamiento básico del equipo, mientras la distribución agrega las herramientas que permiten utilizarlo diariamente.

Puedes ampliar este tema en: ¿Qué es Linux?

Linux y GNU/Linux

Técnicamente, Linux es el núcleo. Muchas distribuciones combinan ese núcleo con herramientas del proyecto GNU y otros programas.

Por esta razón también puedes encontrar el nombre GNU/Linux.

Ventajas de utilizar Linux

  • Cuenta con numerosas distribuciones gratuitas.
  • Puede funcionar en equipos con diferentes capacidades.
  • Permite personalizar gran parte del sistema.
  • Es muy utilizado en servidores.
  • Incluye herramientas útiles para programación.
  • Cuenta con una comunidad amplia.
  • Permite administrar tareas desde la terminal.
  • Ofrece modelos de permisos sólidos.
  • Es importante en computación en la nube.
  • Favorece el aprendizaje sobre sistemas operativos.
No necesitas utilizar solamente la terminal

Las distribuciones modernas incluyen escritorios, ventanas, menús y tiendas de aplicaciones.

¿Qué es una distribución de Linux?

Una distribución es una versión completa del sistema preparada para diferentes necesidades.

Ubuntu

Distribución popular con abundante documentación y herramientas para principiantes.

Linux Mint

Ofrece un escritorio familiar y una experiencia sencilla para comenzar.

Debian

Es reconocida por su estabilidad y sirve como base para otras distribuciones.

Fedora

Integra tecnologías modernas y herramientas orientadas al desarrollo.

Arch Linux

Ofrece mucha personalización, pero requiere más conocimientos técnicos.

Raspberry Pi OS

Está diseñada principalmente para dispositivos Raspberry Pi.

Recomendación para comenzar

Ubuntu o Linux Mint suelen ser opciones accesibles para una primera experiencia.

Formas de probar Linux

Máquina virtual

Ejecuta Linux dentro de otro sistema sin reemplazarlo.

USB en modo live

Permite iniciar Linux desde una memoria USB antes de instalarlo.

Instalación junto a otro sistema

Permite elegir qué sistema utilizar al encender el equipo.

Equipo dedicado

Instala Linux como sistema principal en una computadora.

Crea una copia de seguridad

Antes de modificar particiones o instalar un sistema, guarda tus archivos importantes en otra ubicación.

Escritorio y terminal

Linux puede utilizarse mediante una interfaz gráfica o mediante una terminal.

Entorno de escritorio

El entorno de escritorio incluye ventanas, paneles, iconos, menús y aplicaciones gráficas.

Algunos entornos conocidos son:

  • GNOME.
  • KDE Plasma.
  • Cinnamon.
  • XFCE.
  • MATE.

Terminal

La terminal permite escribir instrucciones para navegar, administrar archivos y ejecutar programas. 

echo "Hola desde Linux"

Este comando muestra un mensaje en la terminal.

No ejecutes comandos que no comprendas

Una instrucción incorrecta puede modificar o eliminar archivos importantes.

Cómo se organiza el sistema de archivos

Linux organiza los archivos a partir de una carpeta principal llamada raíz. 

/
/home

Contiene las carpetas personales de los usuarios.

/etc

Contiene numerosos archivos de configuración.

/var

Guarda datos que cambian, como registros y archivos de servicios.

/usr

Contiene programas, bibliotecas y recursos del sistema.

/tmp

Se utiliza para archivos temporales.

/media

Puede utilizarse para montar dispositivos extraíbles.

Carpeta personal

Cada usuario suele tener una carpeta dentro de /home. 

/home/ana

El símbolo siguiente representa normalmente la carpeta personal del usuario actual: 

~

Comandos básicos de navegación

pwd

Muestra la ubicación actual. 

pwd

ls

Muestra los archivos y carpetas de la ubicación actual. 

ls

Para mostrar más detalles: 

ls -l

Para incluir archivos ocultos: 

ls -a

cd

Permite cambiar de carpeta. 

cd Documentos

Para regresar a la carpeta anterior: 

cd ..

Para volver a la carpeta personal: 

cd ~

clear

Limpia visualmente la terminal. 

clear

Crear y administrar archivos y carpetas

mkdir

Crea una carpeta nueva. 

mkdir proyecto

touch

Puede utilizarse para crear un archivo vacío. 

touch notas.txt

cp

Copia un archivo. 

cp notas.txt copia-notas.txt

Para copiar una carpeta con su contenido: 

cp -r proyecto copia-proyecto

mv

Permite mover o cambiar el nombre de un archivo. 

mv notas.txt apuntes.txt

Para moverlo hacia otra carpeta: 

mv apuntes.txt Documentos/

rm

Elimina archivos.

Utiliza rm con mucho cuidado

Los archivos eliminados desde la terminal pueden no pasar por la papelera. 

rm archivo.txt

Para eliminar una carpeta vacía: 

rmdir carpeta-vacia
Evita copiar comandos destructivos

No utilices opciones recursivas o forzadas sin comprender exactamente qué archivos afectarán.

Visualizar y buscar contenido

cat

Muestra el contenido de un archivo pequeño. 

cat notas.txt

less

Permite leer archivos largos de forma progresiva. 

less registro.txt

head

Muestra las primeras líneas. 

head archivo.txt

tail

Muestra las últimas líneas. 

tail archivo.txt

grep

Busca texto dentro de un archivo. 

grep "Linux" notas.txt

find

Busca archivos y carpetas. 

find . -name "notas.txt"

El punto indica que la búsqueda comienza desde la carpeta actual.

Usuarios y permisos

Linux controla quién puede leer, modificar o ejecutar archivos.

Los permisos básicos son:

r

Permiso de lectura.

w

Permiso de escritura.

x

Permiso de ejecución.

Consultar permisos

ls -l

El resultado puede incluir una estructura similar a: 

-rw-r--r--

chmod

Permite modificar permisos. 

chmod +x script.sh

Este ejemplo agrega permiso de ejecución.

sudo

Permite ejecutar determinadas acciones con privilegios administrativos.

No utilices sudo sin necesidad

Los privilegios administrativos permiten modificar partes importantes del sistema.

Instalar y actualizar programas

Las distribuciones utilizan administradores de paquetes para instalar, actualizar y eliminar programas.

Ubuntu, Debian y Linux Mint

Actualizar la lista de paquetes 
sudo apt update
Instalar actualizaciones 
sudo apt upgrade
Instalar un programa 
sudo apt install nombre-paquete
Eliminar un programa 
sudo apt remove nombre-paquete

Fedora

sudo dnf install nombre-paquete
Utiliza repositorios confiables

Instala programas desde los repositorios oficiales o desde fuentes reconocidas.

Procesos y recursos del sistema

Un proceso es un programa que se está ejecutando.

ps

Muestra información sobre procesos. 

ps

top

Muestra procesos y consumo de recursos en tiempo real. 

top

free

Muestra información sobre memoria. 

free -h

df

Muestra el espacio disponible en los sistemas de archivos. 

df -h

kill

Envía una señal a un proceso.

Identifica correctamente el proceso

Detener un proceso importante puede afectar el sistema o provocar pérdida de información.

Comandos básicos de red

ip address

Muestra las interfaces y direcciones de red. 

ip address

ping

Comprueba si existe comunicación con otro destino. 

ping bytenova.app

Para detenerlo normalmente puedes presionar: 

Ctrl + C

curl

Permite realizar solicitudes y consultar recursos desde la terminal. 

curl https://bytenova.app/

hostname

Muestra el nombre del equipo. 

hostname

Seguridad básica en Linux

  • Mantén el sistema actualizado.
  • Utiliza una contraseña segura.
  • No trabajes siempre como administrador.
  • Instala programas desde fuentes confiables.
  • Revisa los comandos antes de ejecutarlos.
  • Crea copias de seguridad.
  • Elimina servicios que no necesites.
  • Protege las claves de acceso remoto.
  • Revisa los permisos de archivos importantes.
  • Bloquea la pantalla cuando te alejes.
Linux también requiere buenas prácticas

Ningún sistema operativo elimina por completo los riesgos de errores, programas maliciosos o accesos no autorizados.

Software libre y código abierto

Muchas herramientas de Linux se desarrollan bajo licencias abiertas.

Ejercicios para practicar

Explora tu ubicación

Utiliza pwd y ls.

Crea una estructura

Crea una carpeta con varios archivos de prueba.

Copia y mueve archivos

Practica cp y mv.

Busca información

Utiliza grep dentro de un archivo.

Consulta recursos

Revisa memoria y almacenamiento disponibles.

Consulta la red

Utiliza ip address sin publicar información privada.

Práctica guiada

mkdir practica-linux

cd practica-linux

touch notas.txt

echo "Estoy aprendiendo Linux" > notas.txt

cat notas.txt

cp notas.txt copia.txt

ls -l

Este ejercicio crea una carpeta, genera un archivo, escribe una frase, copia el archivo y muestra el resultado.

Practica dentro de una carpeta de prueba

Evita experimentar con carpetas o archivos importantes del sistema.

Preguntas frecuentes

¿Linux es difícil de utilizar?

Las tareas básicas mediante una interfaz gráfica pueden ser sencillas. La terminal requiere práctica progresiva.

¿Linux es gratuito?

Muchas distribuciones pueden descargarse y utilizarse gratuitamente.

¿Necesito borrar mi sistema actual?

No. Puedes comenzar mediante una máquina virtual o una memoria USB en modo live.

¿Linux sirve para programar?

Sí. Incluye o permite instalar numerosas herramientas de programación y desarrollo.

¿Todos los programas funcionan en Linux?

No todos cuentan con una versión para Linux, aunque existen alternativas y herramientas de compatibilidad.

¿Debo memorizar todos los comandos?

No. Aprende los comandos más frecuentes y consulta documentación cuando necesites otros.

Conclusión

Linux es un sistema flexible y muy importante dentro de la programación, los servidores, las redes y la infraestructura tecnológica.

Aprender a navegar por sus carpetas, administrar archivos, instalar programas y consultar recursos proporciona una base sólida para continuar estudiando tecnología.

Comienza con comandos sencillos, practica dentro de carpetas de prueba y aumenta gradualmente la dificultad.
